Setup

Two paths: standard Docker Compose for any Linux host, and Unraid via the Docker manager.

A — Docker Compose

Prerequisites

  • Docker ≥ 24 + Compose plugin
  • An LLM provider: API key or a compatible CLI binary in PATH
  • A Telegram Bot token (from @BotFather ↗)

Installation

1

Clone the repository

git clone https://github.com/mirkofelt/claude-works.git
cd claude-works
2

Start the container

docker compose up -d

On first boot the daemon enters INITIALIZE mode. The Web UI is available immediately.

3

Get the setup token

docker logs claude-works 2>&1 | grep "SETUP TOKEN"

The token is a one-time secret printed to stdout. It is not written to the log file.

4

Complete the setup wizard

Open http://localhost:8080, enter the setup token, and fill in:

  • Telegram bot token + your admin chat ID
  • LLM provider (api with API key, or cli with binary name)
  • Web UI auth token (choose a long random string)

After saving, the daemon auto-transitions to RUN mode within 10 seconds.

Config storage All settings are saved in /data/config.db. No config files to edit — the Web UI is the only interface. Changes take effect within 5 seconds without a restart.

Internet access via Cloudflare Tunnel

To expose the UI securely without opening ports:

  1. Create a tunnel at dash.cloudflare.com → Zero Trust → Networks → Tunnels
  2. Copy the tunnel token
  3. Add a Public Hostname pointing to http://claude-works:8080
  4. Optionally add a Cloudflare Access policy (Zero Trust → Access) to restrict to your identity
  5. Start with the Cloudflare overlay:
CLOUDFLARE_TUNNEL_TOKEN=your_token_here \
docker compose -f docker-compose.yml -f docker-compose.cloudflare.yml up -d

Set web.trusted_proxy: true in config so rate limiting uses real IPs, not Cloudflare's.

No open ports With the Cloudflare overlay, remove or comment out the ports: section in docker-compose.yml so port 8080 is no longer exposed on the host.

B — Unraid

Prerequisites

  • Unraid 6.12+ with Docker enabled
  • Same LLM provider and Telegram token as above

Installation

1

Add container via Docker manager

Unraid UI → Docker → Add Container (Advanced View).

2

Fill in container settings

FieldValue
Nameclaude-works
Repositoryghcr.io/mirkofelt/claude-works:latest or build locally
Network TypeBridge
PortHost: 8080 → Container: 8080
VolumeHost: /mnt/user/appdata/claude-works → Container: /data

Leave all environment variables empty — config goes into the DB via the setup wizard.

3

Start the container and get the setup token

Click Apply. Once running, open the container log in Unraid UI and copy the SETUP TOKEN line.

4

Complete the setup wizard

Navigate to http://<unraid-ip>:8080 and complete setup as in step 4 above.

Internet access on Unraid

Use a second container for the Cloudflare Tunnel:

  1. Add another container: repository cloudflare/cloudflared:latest
  2. Command / Extra Parameters: tunnel --no-autoupdate run
  3. Environment variable: TUNNEL_TOKEN = your tunnel token
  4. Same network as claude-works (bridge), no port mapping needed
  5. Point the tunnel Public Hostname to http://claude-works:8080
appdata path All persistent data (databases, logs, persona, custom scripts) lives in /mnt/user/appdata/claude-works/. Back this directory up to keep your config and history.

Post-Setup

/data customisation

Place files in the data directory to extend the container without rebuilding:

  • requirements.local.txt — extra Python packages installed at every startup
  • init.sh — shell commands run at every startup (apt installs, tool downloads, etc.)
  • persona.md — Markdown persona for the ChiefAgent

All startup output is logged to logs/init.log with timestamps.

Telegram commands

  • /status — show daemon mode, active agents, queue depth
  • /reload_config — force config reload from DB (admin only)
  • /reload_persona — hot-reload persona.md without restart (admin only)
  • /repair <description> — trigger REPAIR mode manually (admin only)
